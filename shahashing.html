Secure Hash Algorithm

About Secure Hash Algorithm

SHA-256 (Secure Hash Algorithm 256-bit) is a cryptographic hash function that takes input data and produces a fixed-size output of 256 bits, commonly represented as a 64-character hexadecimal number. It is a part of the SHA-2 family of hash functions designed by the National Security Agency (NSA).

Approach

To solve the Universal Hashing problem:

  1. Set up initial constants and variables that the algorithm will use. These include predefined constant values and an initial hash value (known as the "initialization vector").
  2. Take the input message and preprocess it. This involves padding the message to a specific length, usually in multiples of 512 bits, and appending the original message length. This step ensures that the message has a suitable format for processing.
  3. Divide the padded message into blocks of 512 bits. For each block, perform a series of logical and arithmetic operations. This is where the core processing of the message takes place, and it involves multiple rounds of operations.
  4. After processing each block, update the hash values. The hash values are modified based on the processed block, and this step is repeated for each block of the message.
  5. Once all blocks are processed, concatenate the final hash values. This results in the 256-bit (32-byte) hash, representing a unique fingerprint of the original input message.
