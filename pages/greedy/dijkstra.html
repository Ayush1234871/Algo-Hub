Dijkstra's Algorithm

Find the shortest paths between nodes in a graph using a greedy approach.

Time Complexity O(V^2) or O(E log V)
Space Complexity O(V)

🎯 About Dijkstra's Algorithm

Dijkstra's Algorithm finds the shortest path from a starting node to all other nodes in a weighted graph. It works by maintaining a set of nodes whose shortest distance from the source is already known, and iteratively picking the node with the smallest unknown distance.

It is a classic example of a Greedy Algorithm because at each step, it makes the locally optimal choice (choosing the unvisited node with the smallest tentative distance).

🚀 The Approach

1

Initialize

Set the distance to the source node to 0 and all other nodes to infinity. Create a set to keep track of visited nodes.

2

Pick Minimum

Pick the unvisited node with the minimum distance from the source. Initially, this is the source node itself.

3

Update Neighbors

For the current node, calculate the distance to its unvisited neighbors. If the calculated distance is less than the known distance, update it.

4

Mark Visited

Mark the current node as visited. A visited node's shortest distance from the source is finalized.

5

Repeat

Repeat steps 2-4 until all nodes are visited.
