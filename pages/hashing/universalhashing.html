Universal Hashing

Minimizing worst-case collisions through randomized selection from a family of hash functions.

Average Case O(1)
Worst Case O(n)
Space O(n)

🎯 About Universal Hashing

Universal hashing is a randomized algorithm that selects a hash function from a carefully designed family of functions. This approach ensures that for any two distinct keys, the probability of a collision is very low (specifically, 1/m, where m is the table size).

By picking a new function at random whenever the hash table is initialized, universal hashing prevents adversaries from choosing keys that would all hash to the same slot, thereby maintaining O(1) average performance regardless of the input.

🚀 The Approach

1

Select Function Family

Identify a "universal" family of hash functions (e.g., functions of the form ((ak + b) mod p) mod m).

2

Random Initialization

When the program starts, randomly pick parameters (like a and b) from the family.

3

Compute Hash

Use the selected function to compute the index for any given key k.

4

Collision Resilience

Enjoy the mathematical guarantee that collisions will be uniformly distributed across the table.
