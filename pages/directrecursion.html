Direct Recursion

The purest form of recursion where a function calls itself directly to solve a problem.

Space O(n)
Base Case Required
Stack LIFO

🎯 About Direct Recursion

Direct recursion occurs when a function calls itself within its own body. It is a powerful programming technique used to solve problems that can be broken down into smaller instances of the same problem.

Every direct recursive function must have two main components: a Base Case (to stop the recursion) and a Recursive Step (to move closer to the base case). Without a base case, the function would call itself indefinitely, eventually leading to a stack overflow.

🚀 The Approach

1

Define Base Case

Identify the simplest scenario where the answer is known immediately (e.g., n == 0 or n == 1).

2

Recursive Call

Call the function from within itself, typically with a modified (smaller) argument.

3

Stack Management

Each call is pushed onto the system stack. Results are popped and combined as the calls return.

4

Combine Results

Merge the returned values from subproblems to form the final solution.
