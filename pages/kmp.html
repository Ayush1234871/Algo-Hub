Knuth-Morris-Pratt (KMP) Algorithm

Search for occurrences of a pattern within a main text string by employing the observation that when a mismatch occurs, the word itself embodies sufficient information to determine where the next match could begin.

Time Complexity O(n + m)
Space Complexity O(m)

🎯 About KMP Algorithm

The Knuth-Morris-Pratt (KMP) string-searching algorithm searches for occurrences of a "word" or "pattern" `W` within a main "text string" `S` by employing the observation that when a mismatch occurs, the word itself embodies sufficient information to determine where the next match could begin, thus bypassing re-examination of previously matched characters.

This algorithm avoids the redundant comparisons found in the naive approach, making it highly efficient, especially when the pattern contains repeating sub-patterns.

🚀 The Approach

1

Preprocess Pattern (LPS Array)

Construct the Longest Proper Prefix which is also Suffix (LPS) array for the pattern. `lps[i]` stores the length of the longest proper prefix of `pattern[0..i]` that is also a suffix of `pattern[0..i]`.

2

Search Process

Iterate through the text. Maintain a pointer `i` for the text and `j` for the pattern.

3

Match Handling

If `text[i] == pattern[j]`, increment both `i` and `j`. If `j` equals the pattern length, a match is found!

4

Mismatch Handling

If a mismatch occurs after some matches (`j > 0`), update `j = lps[j-1]` to avoid redundant checks. If `j == 0`, simply increment `i`.
